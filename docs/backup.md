# Стратегія резервного копіювання

Для мобільного проєкту резервне копіювання стосується вихідного коду, криптографічних сертифікатів та архівів збірок.

## 1. Вихідний код (Source Code)
* **Стратегія:** Постійне зберігання в хмарному репозиторії.
* **Інструмент:** Git + GitHub.
* **Частота:** З кожним комітом (Continuous).
* **Відновлення:** `git clone [URL]` або `git checkout [commit_hash]` для відновлення конкретної версії.

## 2. Сертифікати розробника (Certificates & Provisioning Profiles)
Критично важливі для можливості підписувати та публікувати додаток.
* **Створення копії:** Експорт Distribution Certificate у форматі `.p12` з додатку Keychain Access на Mac разом із паролем.
* **Зберігання:** Безпечний менеджер паролів (наприклад, 1Password) або захищений корпоративний диск.

## 3. Архіви збірок (App Archives)
* **Що зберігається:** Бінарні файли `.xcarchive` (включаючи dSYM файли для розшифровки креш-логів).
* **Частота:** Кожен Production реліз.
* **Місце:** Локально на Build Machine (в Organizer) та резервна копія папки `~/Library/Developer/Xcode/Archives/` на зовнішній диск (через Time Machine).
